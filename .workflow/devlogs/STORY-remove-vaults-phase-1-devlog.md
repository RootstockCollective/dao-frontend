# Devlog: Remove Vault & BTC-Vault — Phase 1

**Date:** 2026-05-20
**Phase:** 1 of 2
**Branch:** feat/remove-vaults

---

## What Was Planned

Delete all directories and files exclusively owned by the vault/btc-vault features, and update supporting shared infrastructure files that referenced them.

## What Was Done

### Directories deleted (exclusive to vault features)
- `src/app/vault/` — USD Vault page, components, hooks, history
- `src/app/btc-vault/` — BTC Vault pages, components, hooks, services, tests
- `src/app/api/vault/` — USD Vault API routes
- `src/app/api/btc-vault/` — BTC Vault API routes (metrics, history, epoch, principal, etc.)
- `src/app/fund-manager/` — Fund Manager page (entirely gated by `btc_vault` feature flag)
- `src/app/fund-admin/` — Fund Admin page (entirely gated by `btc_vault` feature flag)
- `src/shared/hooks/contracts/btc-vault/` — `useReadRbtcVault*`, `usePermissionsManager`, `useReadSyntheticYield`, `useReadRbtcBuffer`
- `src/lib/abis/VaultAbi.ts`
- `src/lib/abis/VaultDepositLimiterAbi.ts`
- `src/lib/abis/btc-vault/` — `RBTCAsyncVaultAbi`, `BufferAbi`, `PermissionsManagerAbi`, `SyntheticYieldAbi`

### Files modified
- **`src/lib/contracts.ts`** — removed `vault`, `rbtcVault`, `buffer`, `permissionsManager`, `syntheticYield` exports and their imports (`getAbi`, `VaultAbi`, vault-related addresses)
- **`src/shared/hooks/contracts/index.ts`** — removed `export * from './btc-vault'`
- **`src/components/MainContainer/sidebars/useFilteredMenuData.ts`** — removed `usePermissionsManager` dependency and role-based filtering (was exclusively used to gate fund-manager/fund-admin vault nav items); simplified to return menu by connection state only
- **`src/app/shared/formatter.ts`** — removed `formatApy` function and `ctokenvault` symbol config (vault-only dead code); removed `VAULT_BASIS_POINTS` and `VAULT_SHARE_DECIMALS` imports

## Discoveries

- `src/lib/abis/btc-vault/` contained `BufferAbi`, `PermissionsManagerAbi`, and `SyntheticYieldAbi` in addition to `RBTCAsyncVaultAbi`. These were used by `contracts.ts` to expose `buffer`, `permissionsManager`, and `syntheticYield` contract objects — all consumed exclusively by the deleted vault/fund-admin code.
- `next-env.d.ts` and `.next/types/routes.d.ts` were missing from the git worktree (auto-generated by Next.js, not committed). Had to copy from main repo to unblock `lint-tsc`.
- `src/app/shared/formatter.ts` was missed in the initial analysis — it imported `VAULT_BASIS_POINTS` and `VAULT_SHARE_DECIMALS` for the `formatApy` function and `ctokenvault` symbol entry.
- `src/components/Breadcrumbs/Breadcrumbs.tsx` was missed in the initial analysis — contained vault breadcrumb entries (cleaned up in Phase 2).
